# abe-manual — 阿部さん専用マニュアル・指示集約

このリポジトリは **個人用** です。`Case-management`（共有）や `staff-abe`（業務報告）とは分け、**自分のルール・手順・テンプレ**を置きます。

## フォルダ構成

| パス | 用途 |
|------|------|
| `CLAUDE.md`（本ファイル） | Cursor / Claude 向けの**全指示の集約**。PDCAの回し方、漏れ確認フロー、判断基準などをここに追記していく。 |
| `テンプレート/` | 日報・広告文案など、**よく使う雛形** |

## 隣接リポジトリ（同一親フォルダ想定）

- `../Case-management/` … 案件管理（共有）
- `../staff-abe/` … 業務報告・FB

案件の中身を変える作業は `Case-management` 側。本リポは **「どう動くか・何を守るか」** に寄せると整理しやすいです。

## PDCA・漏れ確認（追記用メモ）

以下は **運用しながら本ファイルに具体化**してください。

1. **Plan** … 今日／今週の目的と優先順位（どの案件・どの期限か）
2. **Do** … 実行したこと（事実のみ）
3. **Check** … 期待と結果の差、漏れ（一覧・スプシ・メールの突合）
4. **Act** … 次回のルール変更・チェックリスト更新

**漏れ確認フローの例（初期案）**

- MEO：`MEO案件一覧.md` と各 `cases/meo/*.md` の ST・投稿文作成状況の突合
- RB：`RB広告案件一覧.md` と `cases/ads/*.md` の `MTG日時`・`定例会ステータス`
- 送信前：宛名・件名・添付・テンプレ差し込みの**保存前チェック**

## Cursor での参照

ワークスペースに本フォルダを含めるか、ルールで `@abe-manual/CLAUDE.md` を指定すると、チャットから指示を引きやすくなります。

## GitHub への初回接続

ローカルで `git init` 済みの場合、**空のリポジトリ**を GitHub に作成し、次を実行してください。

```bash
cd /path/to/abe-manual
git remote add origin https://github.com/<ユーザー>/<abe-manual>.git
git push -u origin main
```

`staff-abe/All-Push.sh` は `origin` があるときだけ **abe-manual** を push します。
